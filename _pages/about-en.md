## About

I am an **AI-native builder** working across agent workflows, multimodal systems, full-stack AI products, vertical-domain data assets, and applied research.

I am currently pursuing an M.Sc. in Computer Technology at Beijing University of Posts and Telecommunications (BUPT), expected to graduate in 2026. I previously received a B.E. in Internet of Things Engineering from BUPT and a First Class Honours degree from Queen Mary University of London.

My work combines four threads:

- **AI Agents and workflow engineering**: OpenClaw/Codex/Claude collaboration, subagent task decomposition, scheduled cron-agent workflows, Feishu reporting, context compression, and knowledge governance.
- **Multimodal systems**: Any2Any multimodal understanding and generation, MLLM deployment and evaluation, role-conditioned multi-agent multimodal systems, LLM + diffusion coordination.
- **Full-stack AI products**: React/Vite/TypeScript frontend, FastAPI backend, Postgres, payment/order/voucher workflows, AI report generation services, ops dashboards, and deployment verification.
- **Research and data**: vision-language models, training-free test-time augmentation, multimodal benchmarks, vertical-domain QA data pipelines, and structured knowledge extraction.

## Selected Work

### Liuyi AI / FateRipple

I founded and led Liuyi AI, an AI-powered traditional-culture and personalized-report product. The public site is available at [www.liuyi.life](https://www.liuyi.life). The product covers topic-based questionnaires, birth-data input, structured analysis, free preview generation, WeChat/Alipay payment, voucher redemption, full-report generation, and report retrieval.

The engineering stack includes a React 19 + Vite + TypeScript frontend, FastAPI base backend, FastAPI generation service, Postgres, Dulupay payment integration, SKU configuration, ops dashboard, generation-service monitor, voucher scripts, and systemd/nginx/CDN deployment workflows.

### DingMing and YiWorld

DingMing is a vertical-domain Python core library and algorithm platform. It packages deterministic domain logic, report services, prompt templates, streaming/non-streaming report managers, and LLM integration boundaries into a reusable base layer.

YiWorld exposes the core charting and report capabilities as services and applications, including a FastAPI base algorithm service, AWS Lambda + Supabase asynchronous report service, React frontend, and Feishu bot backend.

### OpenClaw Multi-Agent Operations

I use OpenClaw agents as operational engineering resources rather than one-off chat tools. The workflows include subagent task decomposition, scheduled agent execution, skill-based Bilibili content/author discovery, JSONL incremental writes, deduplication, status tracking, Feishu daily reports, and long-history compression into reusable project facts.

This work demonstrates how I design, dispatch, review, and govern agent workflows. I keep the contribution boundary explicit: agents execute many subtasks, while I define objectives, constraints, validation requirements, and reusable knowledge structures.

### BIGAI Any2Any and MAGUS

At the Beijing Institute for General Artificial Intelligence (BIGAI), I worked on Any2Any multimodal understanding and generation. The work involved surveying frontier multimodal models and datasets, deploying and evaluating VILA-U, NeXT-GPT, Wan2.1, and Open-Sora, exploring VAE projectors, constructing multimodal instruction data, and building hard-routed multimodal systems.

This line of work developed into **MAGUS: A Unified Multi-Agent Framework for Universal Multimodal Understanding and Generation**, an arXiv paper on a modular multi-agent framework for text, image, audio, and video understanding/generation. [arXiv:2508.10494](https://arxiv.org/abs/2508.10494)
