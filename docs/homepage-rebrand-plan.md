# Homepage Rebrand Plan

## Goal

将当前基于 academicpages 的个人主页重构为一个 `英文优先、独立中英双版本` 的个人 landing page，核心目标是让访问者在短时间内形成这三个判断：

- 这是一个 `技术型创业者`
- 有 `扎实研究能力`，论文与研究经历是真实硬证明
- 有 `审美、表达和产品判断力`，不是普通学术页，也不是求职简历页

整体风格采用 `极简研究感 + 杂志编辑式编排`，语气 `锋利自信`，视觉改造强度为 `中等偏大胆`，动效仅保留少量精致增强，不做炫技式黑客风。

## Core Positioning

- 移除所有过时的 `seeking PhD`、申请导向和学生求职式叙事。
- 首页主身份切换为 `technical founder / builder / researcher-engineer` 方向的技术型创业者。
- 保留论文、研究经历、代表成果，作为第一层可信度证明。
- 创业身份在首页前两屏明确出现，但暂不展开具体创业项目名称或细节。
- 首页不再按“About / Publications / Awards / Skills”简历顺序平铺，改为一条连续叙事线。

## Homepage Structure

首页改为中长篇叙事式 landing page，建议结构固定为以下顺序：

1. 首屏 Hero
   - 英文优先
   - 杂志编辑式大版心
   - 主标题为一句强定位，不写求职话术
   - 右上角放显著语言切换入口
   - 首屏采用 `卡片式情报面板` 承载信息
   - 保留照片，但弱化处理，不作为视觉主角
2. Identity / Snapshot 模块
   - 用 3-5 个短卡片总结当前身份、研究主轴、构建能力、所在城市/角色状态
3. Research Proof 模块
   - 展示代表论文、研究主题、已发表/在研成果
   - 强调“做过什么问题、提出了什么方法、结果证明了什么”
4. Experience Narrative 模块
   - 将 BIGAI、实验室、创业转型等经历改写为能力演进路径
   - 用“从研究到系统到创业”的线索串起来
5. Selected Signals 模块
   - 精选奖项、机构背书、技能密度、工具栈
   - 只保留高信号内容，删除弱信息和常规简历式废话
6. Contact / Links 模块
   - 收束到对外联系、Scholar、GitHub、邮箱等高价值链接
   - 不保留低价值或空白社交入口

## Bilingual Strategy

- 使用 `独立中英页面`，不是同页文本切换。
- 英文版作为默认入口和主品牌页面。
- 中文版保留同样结构和信息层级，但表达更自然，不做逐句硬翻译。
- 右上角语言切换组件固定可见，保证任一页面都能快速切换。
- 中英文共用同一视觉系统，但允许文案长度和排版细节微调。

## Visual Direction

- 从默认 academicpages 主题中脱离，建立新的首页视觉层级。
- 采用更有辨识度的字体组合与间距系统，目标是研究型高级感，不是通用模板风。
- 颜色策略以克制、高对比、非花哨为原则，强调专业与冷静。
- 使用少量精致动效：
  - 首屏加载渐入
  - 卡片悬停反馈
  - 语言切换与锚点滚动的细微过渡
- 保持移动端完整可读，避免桌面端好看、手机端塌陷。

## Global Cleanup

- 修正站点配置中的低质量占位信息，例如标题、description、author bio 等。
- 清理乱码或编码异常内容，统一中英文文本编码和显示。
- 重新整理导航，改为 `首页强主导 + 少量必要内页`。
- 内页优先保留：
  - Publications / Research
  - About / Bio
  - Contact / Links
- 非核心栏目如 talks、teaching、模板遗留页默认降级或隐藏，除非已有明确价值内容。

## Defaults Chosen

- 英文版为默认主站，中文版为完整镜像式内容版本。
- 本轮不展开创业项目具体名称或商业细节，仅明确创业者身份。
- 头像/照片保留，但只作为辅助身份信号，不做 hero 主图。
- 首页优先承担品牌塑造，论文与研究经历作为最硬证明内容继续保留。
- 默认弱化或隐藏非核心模板遗留栏目，避免信息噪声。
- 文案会同步重写，不保留现有“学生申请型”表述。
